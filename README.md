# DinoRush-C

基于 C++ 与 EasyX 图形库独立研发的 2D 横版动作格斗游戏（单机客户端）。

## 📖 项目简介

DinoRush-C 是一款致敬经典街机（如《恐龙快打》）的 2D 横版清版动作游戏。本项目从零底层构建，完全脱离现有的商业游戏引擎（如 Unity、Cocos 等），仅依赖纯 C++ 与 EasyX 基础图形库实现。

项目总代码量约 3200+ 行，完整实现了从底层游戏主循环（Game Loop）、双缓冲渲染管线，到上层的状态机（FSM）驱动、AABB 物理碰撞检测、基础游戏 AI 以及多关卡流程控制。展示了对 2D 图形学原理、游戏底层架构以及复杂逻辑调度的深入理解与工程落地能力。

---

## 🎮 游戏特性

* **经典街机体验：** 支持八向移动、跑动、普通攻击、踢击等丰富的战斗动作交互。
* **完整的战斗反馈：** 包含精准的攻击命中判定、硬直（受击反馈）、击飞动画以及多阶段的死亡物理表现。
* **多层次关卡设计：** 内置 3 个设计完整的关卡，包含地形边界限制、过场动画（Cutscene）衔接以及难度递增的挑战。
* **多样化敌人 AI：** 包含 3 种普通敌人（具备巡逻、索敌、攻击逻辑）与 1 个最终 Boss（具备高血量、特殊冲撞技能及多段战斗表现）。
* **游戏生态系统：** 实现了完整的玩家 HUD（血量条）、分数统计、场景食物掉落与拾取恢复机制。

## 🛠️ 核心架构与技术实现

### 1. 自定义游戏主循环 (Game Loop)

摒弃了传统应用开发中的事件驱动模型，手工构建了标准的高性能游戏主循环：

* **非阻塞输入轮询：** 使用 `GetAsyncKeyState` 进行高频异步按键检测，支持多键同按（如边跑边跳、斜向移动），提供毫无延迟的操控手感。
* **时间片帧控制：** 依赖 `GetTickCount()` 获取系统高精度运行时间，将**逻辑更新 (Update)** 与 **画面渲染 (Render)** 严格对齐，有效防止因硬件性能差异导致的“游戏加速/减速”问题，保证浮点数坐标和动画帧的平滑过渡。

### 2. 双缓冲渲染与混合渲染管线

针对早期图形库常见的“画面闪烁”与“透明通道丢失”痛点进行了专项优化：

* **双缓冲批绘制 (Double Buffering)：** 利用 `BeginBatchDraw()` 将所有当前帧的精灵图（Sprite）、UI、背景先绘制在内存缓冲区，全部计算完毕后再通过 `FlushBatchDraw()` 一次性推送到显存，彻底消除了画面撕裂。
* **带 Alpha 通道的混合渲染：** 由于 EasyX 默认不支持 PNG 透明通道，本项目底层封装了 Windows GDI 的 `AlphaBlend` API，手动实现了 `transparentimage()` 函数，完美处理了带有透明背景的复杂精灵序列帧渲染。

### 3. AABB 物理碰撞与 2.5D 空间模拟

在纯 2D 坐标系中模拟出了具有深度（Z轴）的街机空间感：

* **基础碰撞检测：** 实现了一套标准的轴对齐包围盒 (AABB) 算法。在攻击判定时，精确计算武器包围盒与敌人受击包围盒的交叉面积，结合角色的朝向（左/右）动态应用伤害。
* **空间地形约束：** 为了模拟 2.5D 视角（角色可以在地面上下移动），代码中抽象出了“碰撞层”。利用线性方程（如 `y >= x * slope + offset`）限制角色的 `(x, y)` 坐标范围，防止角色走出视觉上的“悬崖”或“墙壁”。

### 4. 基于状态机 (FSM) 的动画与逻辑解耦

角色和敌人的行为高度复杂，项目采用状态机模式进行解耦，所有实体均拥有独立的状态枚举（如 `WalkPosture`, `AttackPosture`, `WasKonckedDown` 等）：

* **动画帧隔离：** 不同的状态绑定了不同长度的序列帧（例如：行走 12 帧，攻击 8 帧）。通过帧计数器与时间戳比对，实现毫秒级的序列帧切换。
* **状态互斥与中断：** 严格定义了状态转换条件。例如，在“受击状态 (WasKonckedDown)”播放完成前，强制屏蔽玩家的移动和攻击输入；在“攻击状态”的特定帧（Keyframe）才触发伤害结算，确保“前摇-判定-后摇”的动作连贯性。

### 5. 敌人 AI 行为树系统

为敌人设计了基于距离感知和概率触发的 AI 逻辑：

* **距离感知索敌：** 敌人在每帧计算自身与主角的欧几里得距离。当玩家进入警戒半径时，自动从 `Still`（发呆）切换至 `Track`（追踪）状态；进入攻击半径后触发攻击动作。
* **随机游走策略：** 当玩家脱离仇恨范围时，敌人会触发随机数生成器，在原地驻留与漫游之间无规则切换，使 AI 表现更加自然。
* **Boss 冲撞机制：** 为关底 Boss 单独设计了高速位移逻辑，锁定玩家 Y 轴坐标进行突进，迫使玩家利用 Y 轴走位进行闪避。

## 📂 项目结构

为了追求极致的编译速度与敏捷调试，本项目的全部 3200+ 行核心逻辑内聚于单一入口文件中，实现了零额外配置的极简构建。

```text
DinoRush-C/
├── New Game/
│   └── 源.cpp               # 【核心】游戏主循环、渲染引擎、物理判定、实体逻辑全量代码
├── img/                     # 美术资源目录 (素材版权归原作者所有)
│   ├── Hannah/              # 主角全姿态序列帧图片
│   ├── SmallEnemyA/         # 普通敌人A序列帧
│   ├── SmallEnemyB/         # 普通敌人B序列帧
│   ├── SmallEnemyC/         # Boss 敌人全套动作
│   ├── Background/          # 关卡背景与前景遮罩层
│   ├── Food/                # 恢复类道具图标
│   └── Pterosaur/           # 场景点缀：翼龙动画
├── README.md
├── .gitignore
└── .gitattributes

```

## 🚀 编译与本地运行指南

### 环境依赖

* **操作系统：** Windows
* **编译器：** MSVC (Visual Studio) 或 MinGW
* **第三方库：** [EasyX Graphics Library](https://easyx.cn/) (请提前安装至编译环境)

### 运行步骤 (以 Visual Studio 为例)

1. 打开 Visual Studio，新建一个 **C++ 空项目**。
2. 将当前仓库中的 `New Game/源.cpp` 添加至项目的“源文件”目录中。
3. 将仓库根目录下的 `img/` 文件夹完整拷贝到 VS 生成的 `.vcxproj` 工程文件所在的同级目录下。
4. 确保 Visual Studio 已正确配置 EasyX 库（可通过 EasyX 官方安装程序一键配置）。
5. 直接编译并运行（推荐使用 Release 模式以获得最佳帧率）。

| 键盘按键 | 对应动作 | 说明 |
| --- | --- | --- |
| `W` `A` `S` `D` | **移动** | 控制角色在 2.5D 平面内八向移动 |
